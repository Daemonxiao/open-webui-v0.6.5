# 管理员操作指引：用户注册、用户组管理、模型管理

本文面向平台管理员用户，说明如何在后台完成三类日常管理操作。文中的导航路径按中文界面名称书写，角色枚举值仍保留系统实际值，例如 `admin`、`user`、`pending`。

- 配置新用户注册和审批
- 维护用户组、成员和权限
- 管理模型启用状态、可见性、默认模型和访问范围

截图位置已用“截图占位”标出，发布前请按实际环境补图。

## 1. 前置条件

1. 使用管理员账号登录系统。
2. 进入左侧主界面后，打开 **管理后台**。
3. 后续操作主要使用顶部导航中的 **用户** 和 **设置**。

> [截图占位：登录后进入管理后台的入口位置]

## 2. 用户注册与用户管理

### 2.1 配置是否允许用户自行注册

操作路径：

1. 进入 **管理后台**。
2. 点击顶部导航 **设置**。
3. 在左侧设置菜单中进入 **通用**。
4. 找到 **认证** 区域。
5. 按需要配置以下选项：

| 配置项 | 说明 | 建议 |
| --- | --- | --- |
| **启用新用户注册** | 是否允许用户在登录页自行注册账号 | 对外开放前建议先关闭；需要批量邀请或临时开放时再开启 |
| **默认用户角色** | 新注册用户默认角色，可选 `pending`、`user`、`admin` | 推荐选择 `pending`，由管理员审批后再改为 `user` |
| **默认用户组** | 新用户默认加入的用户组 | 如果已有普通用户组，可以选择该组；否则保持 `None` |
| **在账号待审批页面显示管理员信息** | pending 用户登录后是否看到管理员联系方式 | 推荐开启，并配置有效联系邮箱 |
| **管理员联系邮箱** | pending 用户看到的管理员联系邮箱 | 填写平台管理员或支持邮箱 |
| **待审批用户页面标题 / 内容** | pending 用户等待审批时看到的提示文案 | 可写明审批流程和联系渠道 |

配置完成后，点击页面底部 **保存**。

> [截图占位：设置 > 通用 > 认证 配置区域]

### 2.2 推荐的注册审批流程

建议使用以下流程，避免用户注册后直接获得使用权限：

1. 在 **设置 > 通用 > 认证** 中开启 **启用新用户注册**。
2. 将 **默认用户角色** 设置为 `pending`。
3. 用户自行注册账号。
4. 管理员进入 **用户 > 概览** 查看新用户。
5. 点击用户行中的角色标签或编辑按钮。
6. 在 **编辑用户** 弹窗中将 **角色** 从 `pending` 改为 `user`。
7. 点击 **保存**。

审批后，用户重新登录或刷新页面即可正常使用。

> [截图占位：用户 > 概览 用户列表中 pending 用户位置]
>
> [截图占位：编辑用户弹窗中修改角色]

### 2.3 手动添加单个用户

如果不希望开放注册，可以由管理员直接创建用户：

1. 进入 **管理后台 > 用户 > 概览**。
2. 点击右上角加号按钮 **添加用户**。
3. 在 **表单** 页填写：
   - **角色**：选择 `user`、`admin` 或 `pending`
   - **姓名**：用户姓名
   - **邮箱**：用户邮箱
   - **密码**：初始密码
4. 点击 **保存** 创建账号。
5. 将初始密码通过安全渠道告知用户，并提醒用户首次登录后修改密码。

> [截图占位：添加用户表单]

### 2.4 CSV 批量导入用户

批量创建用户时可使用 CSV 导入：

1. 进入 **管理后台 > 用户 > 概览**。
2. 点击右上角加号按钮 **添加用户**。
3. 切换到 **CSV 导入**。
4. 下载模板文件，或准备包含 4 列的 CSV：

```csv
Name,Email,Password,Role
张三,zhangsan@example.com,InitialPassword123,user
李四,lisi@example.com,InitialPassword123,pending
```

5. 点击上传区域选择 CSV 文件。
6. 点击 **保存** 导入。

注意：

- CSV 列顺序必须是 `Name, Email, Password, Role`。
- `Role` 只能填写 `admin`、`user`、`pending`。
- 批量导入前建议先用 1 条测试数据验证格式。

> [截图占位：CSV 导入上传区域和模板下载位置]

### 2.5 编辑、禁用或删除用户

在 **用户 > 概览** 用户列表中，可以执行以下操作：

- 修改角色：点击用户角色标签或编辑按钮，进入 **编辑用户** 后调整 **角色**。
- 修改用户信息：在 **编辑用户** 中更新姓名、邮箱或头像。
- 重置密码：在 **编辑用户** 中填写 **新密码** 并保存。
- 查看用户所属用户组：**编辑用户** 弹窗中会显示 **用户组**。
- 删除用户：点击用户行右侧删除按钮。

角色说明：

| 角色 | 说明 |
| --- | --- |
| `admin` | 管理员，可进入管理后台管理用户、设置和模型 |
| `user` | 普通用户，可按权限使用聊天、知识库、模型等功能 |
| `pending` | 待审批用户，无法正常使用主要功能 |

## 3. 用户组管理

用户组用于把用户归类，并集中配置权限。建议按部门、团队或使用场景划分，例如：`普通用户组`、`研发组`、`测试组`、`外部试用组`。

### 3.1 创建用户组

操作路径：

1. 进入 **管理后台 > 用户**。
2. 点击左侧或顶部子菜单 **用户组**。
3. 点击右上角 **新建用户组**。
4. 在 **通用** 页填写：
   - **名称**：用户组名称
   - **描述**：用户组说明
   - **谁可以共享到此用户组**：谁可以向该组共享资源
5. 切换到 **权限** 页配置权限。
6. 点击 **保存** 创建用户组。

> [截图占位：用户 > 用户组 列表和新建用户组按钮]
>
> [截图占位：添加用户组 > 通用 页]

### 3.2 配置用户组权限

在用户组弹窗的 **权限** 页，可以配置该组成员可使用的功能。常见权限包括：

| 权限区域 | 常见配置 | 说明 |
| --- | --- | --- |
| **工作区权限** | 模型访问、知识库访问、提示词访问、工具访问、技能访问 | 控制用户能否进入对应工作区功能 |
| **共享权限** | 模型共享、模型公开共享、知识库共享 | 控制用户能否共享模型、知识库等资源 |
| **聊天权限** | 聊天相关操作权限 | 控制普通用户在聊天中的可执行操作 |
| **功能 / 设置** | 功能开关和个人设置能力 | 控制用户可见或可调整的功能 |

建议：

- 普通用户组只开启必要权限。
- **工具访问**、**技能访问** 涉及在服务端运行或上传扩展能力，除非确有需要，否则不要随意开放。
- 如果某项权限来自 **默认权限**，用户组里关闭后可能仍然显示“默认权限仍会启用”。遇到这种情况，需要同时检查默认权限配置。

> [截图占位：用户组 > 权限 页]

### 3.3 添加或移除用户组成员

1. 进入 **管理后台 > 用户 > 用户组**。
2. 点击目标用户组。
3. 切换到 **用户** 页。
4. 在用户列表中搜索目标用户。
5. 勾选 **成员** 列，将用户加入该组。
6. 取消勾选 **成员**，将用户移出该组。

> [截图占位：用户组 > 用户 页中成员勾选列]

### 3.4 配置默认权限

**默认权限** 会应用到所有 `user` 角色用户。配置路径：

1. 进入 **管理后台 > 用户 > 用户组**。
2. 滚动到用户组列表下方。
3. 点击 **默认权限**。
4. 在弹窗中配置普通用户的默认权限。
5. 点击 **保存**。

使用建议：

- 将全员都应该拥有的权限放到默认权限里。
- 将部门或场景差异化权限放到具体用户组里。
- 如果某个用户组关闭某项权限但用户仍然能使用，优先检查默认权限是否仍开启。

> [截图占位：用户组页面底部默认权限入口]

## 4. 模型管理

模型管理主要用于控制系统中有哪些模型可用、是否显示给用户、默认选中哪些模型，以及某个模型对哪些用户或用户组开放。

### 4.1 进入模型管理页面

操作路径：

1. 进入 **管理后台**。
2. 点击顶部导航 **设置**。
3. 在左侧设置菜单中进入 **模型**。

该页面会显示当前所有模型。每个模型行通常包含：

- 模型名称
- 模型 ID 或描述
- `公开` / `私有` 标记
- 编辑按钮
- 更多操作菜单
- 启用/禁用开关

> [截图占位：设置 > 模型 模型列表]

### 4.2 启用或禁用模型

在模型列表中，每个模型右侧有启用/禁用开关：

1. 打开开关：模型处于 **已启用**，用户可使用该模型。
2. 关闭开关：模型处于 **已禁用**，用户不可使用该模型。

批量操作：

1. 点击模型列表右上角更多操作按钮。
2. 选择 **全部启用** 可启用当前筛选范围内的模型。
3. 选择 **全部禁用** 可禁用当前筛选范围内的模型。

> [截图占位：单个模型启用开关]
>
> [截图占位：模型列表更多操作菜单中的全部启用 / 全部禁用]

### 4.3 隐藏或显示模型

隐藏模型用于让模型不在普通选择入口中展示。操作方式：

1. 在模型列表中点击某个模型右侧更多操作按钮。
2. 选择 **隐藏** 或 **显示**。

批量操作：

1. 点击模型列表右上角更多操作按钮。
2. 选择 **全部隐藏** 可隐藏当前筛选范围内的模型。
3. 选择 **全部显示** 可显示当前筛选范围内的模型。

注意：

- **已禁用** 表示模型不可用。
- **已隐藏** 表示模型隐藏展示入口，但不等同于删除模型。
- 如果要彻底限制某个用户或用户组使用模型，应同时检查模型的 **访问权限** 配置。

> [截图占位：单个模型隐藏 / 显示操作]
>
> [截图占位：批量全部隐藏 / 全部显示操作]

### 4.4 使用筛选和搜索定位模型

模型列表支持搜索和视图筛选：

- 搜索框：按模型名称搜索。
- 视图筛选：按全部、启用、禁用、可见、隐藏、公开、私有等状态筛选。

建议在批量启用、禁用、隐藏、显示前，先通过筛选确认当前列表范围，避免误操作。

> [截图占位：模型搜索框和筛选标签]

### 4.5 配置模型访问范围

模型访问范围决定谁可以使用该模型。

操作路径：

1. 在 **设置 > 模型** 中点击目标模型名称或编辑按钮。
2. 进入模型编辑页。
3. 点击右上区域的 **访问权限**。
4. 在访问控制弹窗中配置可访问对象：
   - 公开给所有用户
   - 指定用户组
   - 指定用户
5. 保存配置。
6. 回到模型列表确认模型显示为 `公开` 或 `私有`。

建议：

- 面向所有普通用户开放的模型，可以设为公开。
- 只给某个团队使用的模型，优先授权给用户组，不建议逐个用户授权。
- 测试模型、成本较高模型或敏感能力模型，建议保持私有并只授权指定用户组。

> [截图占位：模型编辑页中的访问权限按钮]
>
> [截图占位：访问权限控制弹窗]

### 4.6 配置默认模型和置顶模型

默认模型会影响新建聊天时自动选择的模型；置顶模型会影响用户侧边栏或模型入口中的优先展示。

操作路径：

1. 进入 **管理后台 > 设置 > 模型**。
2. 点击右上角 **设置**。
3. 在弹窗中进入 **默认值**。
4. 配置：
   - **已选择模型**：新聊天默认选中的模型
   - **置顶模型**：默认置顶展示给用户的模型
   - **提示词建议**：默认提示词建议
   - **模型能力**：默认模型能力
   - **模型参数**：默认模型参数
5. 点击 **保存**。

> [截图占位：模型 > 设置 > 默认值 页]

### 4.7 调整模型展示顺序

操作路径：

1. 进入 **管理后台 > 设置 > 模型**。
2. 点击右上角 **设置**。
3. 切换到 **显示**。
4. 在 **重新排序模型** 中调整模型顺序。
5. 点击 **保存**。

> [截图占位：模型 > 设置 > 显示 页]

### 4.8 导入和导出模型配置

模型列表页支持导入、导出模型配置：

1. 点击 **导出** 导出当前模型配置 JSON。
2. 点击 **导入** 上传模型配置 JSON。

使用建议：

- 大规模调整模型配置前，先导出备份。
- 从其他环境导入配置前，确认目标环境中对应基础模型或连接已存在。
- 导入后检查模型启用状态、访问范围和默认模型配置。

> [截图占位：模型页面导入 / 导出按钮]

## 5. 建议的日常管理流程

### 5.1 新用户入职

1. 管理员创建用户，或允许用户自行注册。
2. 如果用户是自行注册，先保持 `pending`。
3. 管理员审核身份后，将角色改为 `user`。
4. 将用户加入对应用户组。
5. 确认用户组权限和模型访问范围。
6. 通知用户登录验证。

### 5.2 新团队接入

1. 创建团队对应用户组。
2. 配置该组权限。
3. 将团队成员加入用户组。
4. 配置团队可访问模型。
5. 用一个普通用户账号验证实际可见模型和功能入口。

### 5.3 新模型上线

1. 在 **设置 > 模型** 确认模型已出现在列表。
2. 先保持私有或仅授权测试用户组。
3. 测试模型响应、权限、可见性。
4. 需要对全员开放时，再改为公开或授权普通用户组。
5. 如需默认使用，加入 **已选择模型** 或 **置顶模型**。

## 6. 常见问题

### 用户注册后为什么不能使用？

通常是因为 **默认用户角色** 设置为 `pending`。管理员需要进入 **用户 > 概览**，编辑该用户并将角色改为 `user`。

### 用户已经加入用户组，为什么还是看不到某个模型？

请同时检查：

- 模型是否处于 **已启用**。
- 模型是否被 **已隐藏**。
- 模型 **访问权限** 是否授权给该用户组。
- 用户是否确实在该用户组内。

### 关闭用户组权限后，用户为什么仍然有权限？

可能是 **默认权限** 仍然开启了该权限。进入 **用户 > 用户组 > 默认权限** 检查默认权限。

### 应该给用户单独授权模型，还是给用户组授权？

优先给用户组授权。只有临时测试、个别例外或排障场景，才建议给单个用户授权。
